## ðŸš€ Key Technical Challenges Solved

### 1. The "Offline-to-Online" Sync Bridge
**Problem:** The restaurant changes prices locally in the POS, but the public web is hosted on GitHub Pages (static).
**Solution:** I engineered a Python pipeline (`publish_manager.py`) that:
1.  Queries the local SQLite database.
2.  Generates a sanitized `menu.json`.
3.  Optimizes all new images using a Node.js/Sharp subprocess.
4.  Commits and pushes the changes to the remote GitHub repository automatically.
*Result:* The client updates a price on their local tablet, clicks one button, and the world sees the update in 2 minutes.

### 2. High-Performance Image Optimization
**Problem:** High-res food photos were slowing down the mobile menu (`.jpg/png` > 2MB).
**Solution:** Implemented a recursive Node.js script that detects raw images, converts them to high-efficiency `.webp` format, and rewrites all source code references dynamically.
*Result:* Reduced page load size by ~85% and improved Lighthouse performance score.
